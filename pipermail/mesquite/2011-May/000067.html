The iMeshP.h header packaged with mesquite is out of date with respect to
the other ITAPS headers.  This will be fixed in the next release.  To work
around this issue either a) build mesquite without support for the iMeshP
API or b) replace the file itaps/iMeshP.h in the mesquite source with the
updated iMeshP.h from whatever implementation of that API you intend to link
with Mesquite.

- jason
