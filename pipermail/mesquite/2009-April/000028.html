Time for me to chime in with an outside-mesquite answer for this.

Steve, you can do what you want in several ways:
1) Feed the point locations and element connectivity into mesquite 
directly, set the surface points as being fixed, then smooth the 
interior (all this with a wrapper function you'd have to construct which 
would be callable from Fortran)

2) implement your own version of mesquite's mesh access functions which 
will pull the data directly out of your implementation.  If you're not 
conversant in C++, that might be difficult.

3) Read your mesh into MOAB, and hand it to mesquite that way.  This 
method is more complicated to begin with, but is a good longer-term 
solution, esp. if you're wanting to get the mesh interacting with the 
geometry.  However, the picture you sent for the shuttle mesh looked 
like your points are already on the surface.  If that's true and you're 
ok with their location, this option is probably overkill.

Where is your geometry coming from?  Iges, Step, CAD, facets?  I'm 
interested in seeing if any of the tools I work on would help with this 
process.
