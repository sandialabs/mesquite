> Is there some place that can explain what I need to install and where? 
> 

Not that I know of.  You will need implementations of the Mesh, Geom, and
Relation interfaces (e.g. MOAB, CGM, and Lasso).  If using Mesquite 2.0, you
shouldn't need Babel.  You will then need to get your data into these tools.
 To do that, you could use either intermediate file formats (probably the
only option for the geometry data) or direct API calls (might be easiest for
relating the mesh to the geometry.)

So far, I've been assuming that you will use such tools.  Another option is
to implement the subset of those APIs that are necessary for Mesquite on top
of your own code.  For this case, the you shouldn't need any additional code
other than Mesquite and your own.  From the point of view of integrating
with Mesquite, this is just another way to provide Mesquite with callbacks
to interrogate data.  The only difference is that you could implement then
in C rather than C++ (or perhaps even in Fortran*).

- jason


* I don't know enough about Fortran to understand the complexities of this.
 You'll need to do some stuff that is not typical for a Fortran code, such
as passing function arguments by value and working with pointers.
