Hi Colin,

Sorry I missed this email, I was on vacation until yesterday.  It is
certainly possible to use mesquite in concert with with a F77 ale solver.
However, we have no pre-built wrappers to expose the C++ layer to the
fortran code.  

For an example of a bare minimum mesh optimizer look at:
testSuite/wrapper_tests/shape_improver.cpp

The ArrayMesh implementation of the Mesh interface is designed to handle
fortran data structures so that would likely be the easiest way to inform
Mesquite about your mesh specifics.  You would need to replace the
MeshImpl in that example with an ArrayMesh and pass in pointers to the
fortran data.


In terms of curved boundaries, if there is an analytic representation of
the boundary then the solution is pretty easy and already exists in
Mesquite.  For curved boundaries only defined by the original nodal
positions, I have some local code that may be useful to you for
constraining nodes.  Let me know if you're interested.

Brian
