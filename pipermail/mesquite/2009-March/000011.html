> capabilities of these functions, and the inputs to the functions. The
> wrapper that is provided seems to work for a specific problem. Based on

> This is why I'm so interested in a FORTRAN API. With a FORTRAN API, the
> language barrier is gone and I focus my energy on determining object

Mesquite is fairly object-oriented.  It isn't functions and arguments you
need to know about to construct custom optimizers, but rather classes and
how to assemble them.  A FORTRAN API for this would be a lot of work given
Mesquite's object-oriented implementation.
