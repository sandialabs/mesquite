Hello, 
my name is Tomas Janak, I am a developer of an open source software for positioning and personalization of finite element human body models called PIPER ( [ http://piper-project.org/ | http://piper-project.org/  ] ). We have integrated the Mesquite library from Trilinos into PIPER, but currently we are utilizing only a small subset of its features. We would like to add some new smoothing functionality, so we have been wondering if we can get what we want from Mesquite. But so far I have not found a way to do what I want. 

Here is the idea: our models consist of many topologically disconnected components (representing individual anatomical entities of the body like organs, muscles etc.), but there are generally very small gaps between them. Currently, when we do smoothing of "3D" elements (tetrahedrons, hexahedrons etc.), we fix the vertices on the boundary of each component in order to avoid the components penetrating each other after the smoothing. However, in some cases it is those surface vertices that "are wrong" and so we would like to allow them to move, while still respecting the overall shape of the boundary, so that penetrations do not appear (i.e. we want them to "slide" over the boundary). Does Mesquite provide some tools to achieve that? I was looking at the Settings::set_fixed_vertex_mode(FixedVertexMode) method for ShapeImprover, I see one possible option is FIXED_SURFACE, but simply switching to that does not seem to achieve what we want... 
Note that I do realize that since we are working with discrete meshes, moving the vertices on the boundary will change the shape of it to a certain extent even if their position is on the original boundary (i.e. on some face). But we would still like to try this, hoping we can minimize the changes enough to avoid penetrations. 

Best Regards, 
Tomas Janak
